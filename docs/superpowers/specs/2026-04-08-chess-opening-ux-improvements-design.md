# 国际象棋开局学习游戏 — UX 改进设计文档

**日期：** 2026-04-08
**状态：** 已确认

## 背景

当前项目是一个纯前端单文件 HTML 应用（index.html，~1500 行），包含 25+ 个国际象棋经典开局的知识库和四种学习模式（开局百科、跟走练习、开局问答、猜开局名）。

用户反馈的核心问题：
1. 棋子视觉辨识度差——Unicode 字符白棋在浅色格子上几乎不可见
2. 走错时缺乏教学价值——只闪红提示"错了"，没有解释和后果演示
3. 练习模式需要双方都手动走，应该只走开局方

## 改进 1：SVG 棋子系统

### 方案
用内联 SVG 替换现有 Unicode 字符棋子。

### 设计细节
- 在 `<script>` 顶部定义 `PIECE_SVG` 对象，包含 12 个棋子（黑白各 6 种：K/Q/R/B/N/P）的 SVG path 数据
- 采用 Colin M.L. Burnett 经典棋子设计（公共领域许可，lichess 同款）
- 白棋样式：`fill: #fff, stroke: #333, stroke-width: 1.5`
- 黑棋样式：`fill: #333, stroke: #333, stroke-width: 1.5`，细节用 `stroke: #fff` 区分
- SVG viewBox 统一 `0 0 45 45`
- `renderBoard()` 函数改为生成 `<svg>` 元素而非文本节点
- 棋子尺寸跟随格子：正常 60px → SVG 宽高 44px，移动端 42px → SVG 宽高 30px

### 不改的
- 棋盘颜色（`#f0d9b5` / `#b58863`）保持不变
- 棋盘布局、坐标标注、高亮/闪烁动画逻辑不变

## 改进 2：走错后的教学反馈

### 方案
复用左侧语音泡泡组件，走错时显示红色警示泡泡 + 后果演示动画。

### 触发时机
玩家在练习模式中点击了错误的目标格子时（现有 `onPracticeClick` 函数内）。

### 交互流程
1. 格子闪红（保留现有 `wrongFlash` 动画）
2. 左侧泡泡变红弹出，显示：
   - 红色标题 "❌ 白方"（或黑方）
   - 解释文字：为什么正确走法更好
   - "👁 看看后果" 按钮
3. 点击"看看后果" → 棋盘自动演示 2-3 步惩罚走法，每步间隔 800ms，带棋子移动动画
4. 演示完毕 → 泡泡内容变为总结文字 + "↩ 回到正确位置" 按钮
5. 点击回退 → 棋盘恢复到走错前的状态，玩家重新走

### 数据结构
在 OPENINGS 数组的每个开局对象中新增 `mistakes` 字段：

```js
mistakes: [
  {
    atMove: 4,                              // 在第几步可能走错（0-indexed）
    wrong: "f1b5",                          // 错误走法
    explain: "Bb5 钉住了马，但没有直接威胁。Bc4 瞄准 f7 弱点更有攻击性。",
    punishment: ["d7d5", "b5c6", "b7c6"],   // 对手惩罚走法序列
    punchline: "黑方用 d5 反击中心，白象被迫交换，黑方获得双象优势"
  }
]
```

### 覆盖范围
所有 25+ 个开局都要有 mistakes 数据，每个开局至少 1-2 个常见错误。

### 泡泡样式变化
- 正常提示：现有的金色（白方）/ 蓝色（黑方）边框
- 走错提示：红色边框（`border-color: #f44336`），背景 `rgba(60,20,20,0.95)`
- 新增 CSS 类 `.speech-bubble.error-bubble`

### 没有匹配到预设错误时的行为
如果玩家的错误走法不在 `mistakes` 列表中，只显示通用泡泡："这步不是主线走法。正确走法是 XX" + 对应的 `moveExplain` 文字，不显示"看看后果"按钮。

## 改进 3：对手自动走

### 方案
练习模式中，对手的走法自动执行，玩家只操作开局方。

### 设计细节
- OPENINGS 数据中新增 `playerSide` 字段：`"white"` 或 `"black"`
- 判断规则：名字含"防御"或"Defense"的开局 → `playerSide: "black"`（玩家扮演黑方学防御），其余 → `playerSide: "white"`
- 对手走法在 500ms 延迟后自动执行
- 自动走时带棋子滑动动画（CSS transition，棋子从起始格滑到目标格）
- 自动走的同时，对应方的泡泡显示该步的 `moveExplain` 解释
- 自动走期间禁用棋盘点击，防止误操作

### 不影响的模块
- 开局百科、问答、猜题三个模式不受影响

## 改进 4：棋步导航控制

### 方案
在棋盘下方添加前进/后退导航，支持键盘快捷键。

### 设计细节
- 棋盘下方进度条旁添加 `←` `→` 按钮
- `←`：回退一步（撤回上一个走法，包括自动走的对手步）
- `→`：前进一步（如果之前回退过，可重新前进；若是对手步则自动执行）
- 回退到某一步后，玩家可从该位置重新走（走出新步后，后续历史清除）
- 键盘左右方向键（`ArrowLeft` / `ArrowRight`）绑定同样功能
- 按钮样式与现有 `btn-secondary` 一致

### 实现要点
- 维护一个 `moveHistory` 数组和 `currentMoveIndex` 指针
- 回退时从 `moveHistory` 重新计算棋盘状态（从初始局面重放到目标步数）
- 前进时从 `moveHistory` 取出下一步执行

## 技术约束

- 保持单文件结构（index.html）
- 零外部依赖
- 移动端响应式不破坏
- 现有的开局百科、问答、猜题功能不受影响
